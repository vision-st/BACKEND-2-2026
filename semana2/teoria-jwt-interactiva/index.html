Conceptos clave de la guía

Autenticación

Responde a la pregunta: ¿quién eres?

Ejemplo: el usuario envía credenciales y el sistema valida su identidad antes de entregar un token.

Autorización

Responde a la pregunta: ¿qué puedes hacer?

Ejemplo: un usuario autenticado puede ver su perfil, pero solo un admin puede entrar al panel de administración.

Stateful vs Stateless

Autenticación con estado (Stateful)

  • El servidor guarda la sesión del usuario.
  • Cada solicitud depende de una sesión previa.
  • Es cómoda para aplicaciones tradicionales con formularios.
  • Escala peor en arquitecturas distribuidas.

Estructura de un JWT

Header
Payload
Signature

Header

Indica tipo de token y algoritmo de firma. Ejemplo: { "alg": "HS256", "typ": "JWT" }

Payload

Contiene claims como usuario, roles, emisor o expiración. Ojo: está codificado, no cifrado.

Signature

Permite verificar que el token no fue alterado y que fue emitido por quien corresponde.

Buenas prácticas con JWT

Haz esto

  • Usar HTTPS.
  • Definir expiración del token.
  • Proteger la clave secreta.
  • Validar firma y claims críticas.

No hagas esto

  • Guardar contraseñas en el payload.
  • Usar tokens eternos.
  • Confiar en JWT sin validar firma.
  • Enviar tokens por HTTP plano.
